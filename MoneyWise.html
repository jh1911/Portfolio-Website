MoneyWise: a money saving tool

Introduction

  • Duration: 2 Months
  • Tools: Pen & Paper, Figma, Miro, Adobe Photoshop
  • Objective: Creating a money saving tool for people that don't like to spend a lot of time with their finances but want to reach their financial goals and save money.

The Process

The Design Process for this proudct started with ideation, where the main goal was to create a money saving tool that not only looked aesthetically pleasing but also helped users achieve their goals.

After defining the main idea, competitor research was conducted to identify what was already on the market and to gather insights.

Problems to solve with this Objective in Mind were:

  1. The Target User Designing an app for users who don't like to spend time with their finances.

  2. Usability & Complexaty Desinging an app for users which is easy to use while incorporating complex user flows.

How I solved Problem #1

Creating a brand which values Simplicity, Trust, Inclusivity & Empowerment

To make a finance app appealing to users who don't like to spend time with their finance I had to develope a communicative brand, which spreads Simplicity, Trust, Empowerment & Inclusivity.

To get to this goal I:

  1. Developed a brand
  2. Created a style guide

From Low-Fidelity to Mid-Fidelity

After the brand was created I sketched out Low-Fidelity Wireframes to plan out how the app would function. From there on greyscale Mid-Fidelity prototypes were created to conduct user tests.

User Feedback:

Based on the feedback received the final UI design and mockups were created. This iterative approach ensured that the product meets the users' needs, was easy to use & looks aesthetically pleasing.

Fixed whitespacing and polished the visual hierarchy.

From there on I applied the UI design to every screen

Final UI Design

What I have learned from this project

  1. creating a thought-out style guide can only benefit the product. It sets a really solid foundation for the UI design.
  2. simplifying the features of an app can greatly benefit the user by allowing them to accomplish their goals more efficiently.
  3. User testing is key. Through that I was able to identify and eliminate pain points customers could later encounter.
